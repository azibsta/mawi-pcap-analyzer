## System Overview

The system is designed to process extremely large PCAP (Packet Capture) trace files from the MAWI dataset, analyze protocol trends (TCP, UDP, IPv6, QUIC, HTTP/HTTPS) over multiple decades, and present the data visually.

It consists of two main components:
1. **The Backend (`analyze.py`)**: A powerful Python CLI that orchestrates a high-performance C++ engine (`mawi_engine.cpp`) to parse PCAP files, compute statistics, and cache the results into efficient Parquet files. It exports the final results as JSON or CSV files.
2. **The Frontend (`frontend/serve.py`)**: A dynamic, glassmorphic web dashboard that reads the JSON exports to generate interactive charts and tables.

## CLI Usage

The CLI is used to trigger the parsing of PCAP files and generate the reports. Open your terminal (PowerShell or Command Prompt) and navigate to the project directory:
```powershell
cd "C:\Users\User\Desktop\FYP\new analyzer"
```

### Basic Commands

**Analyze a single year:**
```powershell
python analyze.py --year 2015
```

**Analyze a range of years:**
```powershell
python analyze.py --from 2006 --to 2025
```

### Exporting Reports (For the Frontend)
To view data on the dashboard, you *must* export the results to a JSON file. The dashboard automatically looks for `reports/trends_2006_2025.json`.
```powershell
python analyze.py --from 2006 --to 2025 --export json
```
*(You can also use `--export csv` or `--export both`)*

### Advanced Options

*   **Force Re-parsing (`--no-cache`)**: The system automatically caches parsed data as `.parquet` files for blazing-fast subsequent runs. If you add new data or modify the parsing logic, use this flag to ignore the cache.
    ```powershell
    python analyze.py --from 2006 --to 2025 --no-cache
    ```
*   **Compile C++ Engine (`--build`)**: If you make changes to `mawi_engine.cpp`, run this flag to recompile the high-performance parser before running the analysis.
    ```powershell
    python analyze.py --year 2015 --build
    ```
*   **Limit Packet Count (`--max-packets N`)**: Useful for quickly testing logic on massive files without waiting for the whole file to parse.
    ```powershell
    python analyze.py --year 2018 --max-packets 1000000
    ```
*   **Generate Milestones (`--milestones`)**: Prints out historic milestone events (e.g., "HTTPS overtakes HTTP") detected in the data.

## Frontend Usage

The frontend dashboard provides a beautiful, interactive interface for analyzing the JSON reports generated by the CLI. Because of modern browser security policies (CORS), the dashboard cannot simply be double-clicked to open. It must be hosted on a local web server.

### Launching the Dashboard

1. Open your terminal and navigate to the **frontend** folder:
   ```powershell
   cd "C:\Users\User\Desktop\FYP\new analyzer\frontend"
   ```
2. Run the server script:
   ```powershell
   python serve.py
   ```
3. The script will automatically start a local server and pop open your default web browser to:
   `http://localhost:8000/frontend/index.html`

### Dashboard Features

Once the dashboard is open, it will automatically load the latest `trends_2006_2025.json` file from your `reports` directory.

*   **Interactive Charts**: Hover over any data point to see exact metrics. Click on any data point in the line or bar charts to automatically sync the "Top Ports" pie chart to that specific year!
*   **Theme Toggle**: Click the "☀️ Light Mode / 🌙 Dark Mode" button to swap the dashboard's color palette.
*   **Export to PDF**: Click the "📄 Export PDF" button to download a high-resolution snapshot of the entire dashboard—perfect for inserting directly into your FYP thesis!
*   **Raw Dataset Table**: Scroll to the bottom to view the raw metrics in a sortable, paginated table.

## File Structure Reference

*   `analyze.py` - The main Python CLI program.
*   `pipeline.py` - The core logic linking Python to the C++ parser and Pandas dataframe caching.
*   `mawi_engine.cpp` - The high-performance C++ packet parsing engine.
*   `data/` - Directory where your raw `.pcap` or `.pcap.gz` files should be placed (organized by year folders).
*   `reports/` - Directory where `trends_*.json` and CSV files are exported.
*   `cache/` - Directory where `.parquet` cache files are stored.
*   `frontend/index.html` - The HTML/Tailwind/Chart.js web dashboard.
*   `frontend/serve.py` - The lightweight web server to launch the dashboard.
